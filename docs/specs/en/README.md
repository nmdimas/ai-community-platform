# Interface And Protocol Specs

This directory contains separate requirements and protocol definitions for the key platform interfaces that must be understood consistently by the core-platform, UI layers, and agent modules.

## Purpose

These documents define:

- what the `web`, `admin`, `api`, and `a2a` layers must do
- which contracts are mandatory
- what AI agents and developers must understand before implementation

## Documents

- `web-requirements.md` — requirements for the public web layer
- `admin-requirements.md` — requirements for the admin layer
- `api-protocol.md` — API contract, endpoint rules, and mandatory OpenAPI documentation
- `a2a-protocol.md` — baseline agent-to-agent protocol for internal agent collaboration

## Language Rule

- `*.md` — Ukrainian canonical version
- `*.en.md` — English mirror
