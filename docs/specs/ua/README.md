# Специфікації Інтерфейсів Та Протоколів

Ця папка містить окремі вимоги та протокольні описи для ключових інтерфейсів платформи, які мають однаково розуміти core-platform, UI-шари та агентні модулі.

## Призначення

Ці документи фіксують:

- що саме повинні робити `web`, `admin`, `api` та `a2a` шари
- які контракти вважаються обов'язковими
- які вимоги мають знати AI-агенти та розробники перед реалізацією

## Документи

- `web-requirements.md` — вимоги до public web-шару
- `admin-requirements.md` — вимоги до admin-шару
- `api-protocol.md` — API-контракт, вимоги до endpoint-ів і обов'язкова OpenAPI-документація
- `a2a-protocol.md` — базовий agent-to-agent протокол для внутрішньої взаємодії агентів

## Правило Мови

- `*.md` — канонічна українська версія
- `*.en.md` — англійське дзеркало
