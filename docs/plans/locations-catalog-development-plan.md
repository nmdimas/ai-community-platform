# Development Plan: Locations Catalog

## 1. Goal

Запустити агент, який дозволяє модераторам зберігати корисні рекомендації про місця та сервіси в структурований каталог і знаходити їх через пошук.

## 2. Dependencies

- platform command router
- таблиця `messages`
- нова таблиця `locations`
- full-text search

## 3. Work Breakdown

### Phase 1: Design

- [ ] Уточнити, що вважається "локацією": місце, сервіс, контакт чи всі три типи
- [ ] Узгодити мінімальний набір полів: `name`, `description`, `tags`, `address_text`, `contact_text`
- [ ] Визначити формат команд для додавання і пошуку

### Phase 2: Data

- [ ] Описати схему таблиці `locations`
- [ ] Додати індекси для пошуку по назві, опису і тегах
- [ ] Додати статус запису (`approved` або `draft`, якщо знадобиться)

### Phase 3: Implementation

- [ ] Реалізувати handler для `/location add`
- [ ] Реалізувати handler для `/location <query>`
- [ ] Додати парсинг reply-повідомлення в структурований запис
- [ ] Підключити логування створення і пошуку

### Phase 4: Validation

- [ ] Перевірити збереження реальної рекомендації з чату
- [ ] Перевірити пошук по назві і тегу
- [ ] Перевірити сценарій із порожнім або нечітким повідомленням

## 4. Risks

- у вільному тексті часто бракує структури для якісного каталогу
- без правил модерації каталог може швидко засмітитися
- без геоданих частина пошуків буде неточною

## 5. Exit Criteria

- модератор може зберегти рекомендацію в каталог із чату
- учасник може знайти релевантну локацію командою
- каталог не створює дублюючого шуму в стрічці
