# Development Plan: News Digest

## 1. Goal

Запустити агент, який вручну генерує короткий digest за останній період і публікує його в чат.

## 2. Dependencies

- platform command router
- доступ до `messages`
- таблиця `digests`

## 3. Work Breakdown

### Phase 1: Design

- [ ] Визначити дефолтний часовий діапазон
- [ ] Узгодити формат digest-повідомлення

### Phase 2: Data

- [ ] Описати схему `digests`
- [ ] Додати спосіб не дублювати однакові digest-и

### Phase 3: Implementation

- [ ] Реалізувати `/digest now`
- [ ] Зібрати повідомлення за період
- [ ] Додати просту дедуплікацію
- [ ] Зберігати digest у БД

### Phase 4: Validation

- [ ] Перевірити генерацію на чаті з реальними повідомленнями
- [ ] Перевірити, що digest не містить службових команд
- [ ] Перевірити поведінку при малій кількості даних

## 4. Risks

- низька якість стислого підсумку без додаткової семантики
- дублювання схожих тез при простих правилах

## 5. Exit Criteria

- адмін може вручну згенерувати digest
- digest зберігається і не створює спам
