# PRD: Knowledge Extractor / Community Wiki

## 1. Summary

Агент перетворює корисні повідомлення з чату на структуровані knowledge entries, щоб повторювані відповіді не губились у стрічці.

## 2. Goal

Дати модераторам і учасникам мінімальний wiki-шар поверх чату:

- вручну зберігати цінні повідомлення
- знаходити знання через пошук
- зменшити повтори однакових питань

## 3. Users and Jobs-to-be-Done

- `moderator`: зберегти корисне повідомлення як knowledge entry
- `member`: знайти відповідь через `/wiki <query>`
- `admin`: мати видимий ефект від платформи вже в першому пілоті

## 4. Scope

### In Scope

- ручне збереження knowledge через reply-команду
- full-text пошук по knowledge
- коротка відповідь у чат із топ-результатами

### Out of Scope

- повністю автоматичне витягування знань без модератора
- складне редагування статей
- векторний пошук на старті

## 5. Inputs

- `command.received`
- `message.created` (тільки для reply context)
- таблиця `messages`

## 6. Outputs

- запис у `knowledge`
- chat-відповідь із підтвердженням збереження
- chat-відповідь із результатами пошуку

## 7. UX / Commands

- `/wiki add` у reply на повідомлення
- `/wiki <query>`

## 8. Data Model Usage

### Reads

- `messages`
- `knowledge`

### Writes

- `knowledge(title, body, tags, source_msg_id, created_by, created_at)`

## 9. Rules / Heuristics

- зберігати тільки текстові повідомлення
- якщо команда не у reply, повертати інструкцію
- title формується з перших змістовних слів або вводиться пізніше
- у відповідь на пошук повертати 3-5 найрелевантніших збігів

## 10. Failure Modes

- якщо reply target недоступний, повернути зрозуміле повідомлення
- якщо пошук не дав результатів, показати "нічого не знайдено"
- якщо база недоступна, логувати помилку та не ламати чат-флоу

## 11. Success Metrics

- кількість збережених knowledge entries за тиждень
- кількість викликів `/wiki`
- частка запитів, що завершились корисною відповіддю
