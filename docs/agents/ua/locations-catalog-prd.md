# PRD: Locations Catalog

## 1. Summary

Агент створює структурований каталог локацій, сервісів і перевірених контактів прямо поверх чату, щоб корисні рекомендації не губилися в переписці.

## 2. Goal

Дати ком'юніті простий спосіб:

- зберігати перевірені місця та сервіси
- швидко знаходити рекомендації за запитом
- відділяти корисні рекомендації від випадкового шуму в чаті

## 3. Users and Jobs-to-be-Done

- `moderator`: зберегти корисну рекомендацію з чату в каталог
- `member`: знайти локацію або сервіс за назвою, тегом чи контекстом
- `admin`: сформувати базову "карту" перевірених місць для ком'юніті

## 4. Scope

### In Scope

- ручне додавання локації або сервісу через reply-команду
- текстовий пошук по каталогу
- базові теги для категоризації
- статус `approved` для контрольованого каталогу

### Out of Scope

- мапи та геокоординати
- складний рейтинг або рев'ю-система
- автоматичне витягування всіх рекомендацій з чату без модератора

## 5. Inputs

- `command.received`
- `message.created` (для reply context)
- таблиця `messages`
- таблиця `locations`

## 6. Outputs

- запис у `locations`
- відповідь у чат із підтвердженням збереження
- відповідь у чат із результатами пошуку

## 7. UX / Commands

- `/location add` у reply на повідомлення з рекомендацією
- `/location <query>`

## 8. Data Model Usage

### Reads

- `messages`
- `locations`

### Writes

- `locations(name, description, tags, address_text, contact_text, source_msg_id, status, created_by, created_at)`

## 9. Rules / Heuristics

- зберігати лише текстові повідомлення або повідомлення з текстовим описом
- команда `/location add` працює тільки в reply
- перший MVP зберігає запис у статусі `approved`, якщо дію робить `moderator` або `admin`
- назва локації формується з повідомлення або уточнюється пізніше вручну
- у пошуку повертати 3-5 найбільш релевантних результатів
- теги використовуються для простого фільтрування: наприклад `cafe`, `service`, `coworking`, `repair`

## 10. Failure Modes

- якщо команда викликана не у reply, агент повертає коротку інструкцію
- якщо повідомлення не містить достатньо даних, агент просить уточнити опис
- якщо по запиту немає збігів, повертається "нічого не знайдено"
- якщо база недоступна, помилки логуються, а чат не блокується

## 11. Success Metrics

- кількість збережених локацій/сервісів за тиждень
- кількість пошукових запитів до каталогу
- частка записів, які реально використовуються повторно
- кількість повторних рекомендацій, що перестали дублюватися в чаті
