# A2A Protocol — Теми

> Завантажено з [https://a2a-protocol.org/latest/topics/what-is-a2a/](https://a2a-protocol.org/latest/topics/what-is-a2a/) 2026-03-06.
> Згенеровано скілом `web-to-docs`.

## Сторінки

- [**Що таке A2A?**](./what-is-a2a.md)
  Протокол A2A — відкритий стандарт для безперешкодної комунікації між AI-агентами, що руйнує традиційні бар'єри у розгортанні агентів...

- [**Основні концепції та компоненти**](./key-concepts.md)
  A2A використовує набір базових концепцій — Agent Card, Task, Message, Part, Artifact — які визначають взаємодію між агентами...

- [**Виявлення агентів**](./agent-discovery.md)
  Для співпраці через A2A агенти мають знаходити один одного через Agent Card. Стратегії: well-known URI, реєстри, пряма конфігурація...

- [**Корпоративна реалізація**](./enterprise-ready.md)
  A2A спроектовано з урахуванням корпоративних вимог — TLS, OAuth2/OIDC автентифікація, гранулярна авторизація, приватність даних, трейсинг...

- [**Життєвий цикл Task**](./life-of-a-task.md)
  Взаємодії можуть бути stateless Message або stateful Task з визначеним життєвим циклом. Task підтримує уточнення, паралельні запити та версіювання артефактів...

- [**Розширення (Extensions)**](./extensions.md)
  Extensions дозволяють розширювати A2A новими даними, RPC-методами та стейт-машинами. Агенти декларують підтримку в Agent Card, клієнти активують через HTTP-заголовки...

- [**Стрімінг та асинхронні операції**](./streaming-and-async.md)
  A2A підтримує Server-Sent Events для стрімінгу в реальному часі та push-нотифікації через вебхуки для довготривалих задач...

- [**Порівняння A2A та MCP**](./a2a-and-mcp.md)
  A2A забезпечує міжагентну співпрацю, MCP з'єднує агентів з інструментами. Вони доповнюють одне одного — A2A для партнерства, MCP для використання можливостей...
