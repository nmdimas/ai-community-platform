# Index Templates

This skill generates **two levels** of README files following the project's
bilingual documentation convention (`ua/` + `en/` subfolders).

## 1. Root Section README: `<output_dir>/README.md`

```markdown
# <Site or Section Title>

> Source: [<starting URL>](<starting URL>)
> Fetched: <YYYY-MM-DD> | Generated by `web-to-docs`

## Languages

- [English](./en/README.md)
- [Українська](./ua/README.md)

## Language Rule

- `ua/` — канонічна українська версія
- `en/` — English version
```

### Rules for Root README

1. **Title**: Use the first `# ` heading of the starting page, or the domain name.
2. **Languages section**: List only languages that actually have content.
   If only one language was fetched, still include the section for consistency.
3. **Language Rule**: Always include this block — it matches the convention used
   in `docs/specs/`, `docs/agents/`, `docs/product/`.

## 2. Language-Specific README: `<output_dir>/<lang>/README.md`

```markdown
# <Site or Section Title>

> Fetched from [<starting URL>](<starting URL>) on <YYYY-MM-DD>.
> Generated by the `web-to-docs` skill.

## Pages

- [**<Page Title>**](./<filename>.md)
  <Excerpt: first 1-2 sentences, max ~150 characters>

- [**<Page Title>**](./<filename>.md)
  <Excerpt: first 1-2 sentences, max ~150 characters>
```

### Rules for Language README

1. **Title**: Same as root — first `# ` heading of the starting page,
   or the domain name.

2. **Page order**: Starting page first, then remaining pages in the order
   they appeared as links on the starting page. Pages discovered at depth 2
   appear after all depth-1 pages, grouped under their parent.

3. **Page title**: First `# ` heading in the converted Markdown. Fallback:
   link text from the parent page. Last resort: filename without extension.

4. **Excerpt**: First non-heading, non-empty paragraph from the Markdown.
   Truncate at ~150 characters on a word boundary, append "..." if truncated.

5. **Depth-2 grouping** (when applicable):

   ```markdown
   ## Pages

   - [**Introduction**](./introduction.md)
     Overview of the FooBar framework...

   - [**Getting Started**](./getting-started.md)
     How to install and configure FooBar...

     - [**Installation Details**](./installation-details.md)
       Step-by-step installation for Linux, macOS, and Windows...
   ```

   Sub-pages discovered from a depth-1 page are indented under their parent.

## Example Output Structure

```
docs/fetched/a2a-protocol-org/
├── README.md                  ← Root: links to language versions
└── en/
    ├── README.md              ← English index with page list
    ├── what-is-a2a.md
    ├── key-concepts.md
    ├── agent-discovery.md
    └── ...
```

For bilingual sources:

```
docs/fetched/some-site/
├── README.md
├── en/
│   ├── README.md
│   ├── intro.md
│   └── ...
└── ua/
    ├── README.md
    ├── intro.md
    └── ...
```
