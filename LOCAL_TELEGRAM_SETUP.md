# Локальне налаштування Telegram Бота з OpenClaw

Цей тимчасовий гайд крок за кроком описує, як налаштувати Telegram-бота, щоб він працював з вашим локально розгорнутим OpenClaw, і як дійти до першого тестового повідомлення.

---

## Крок 1. Створення бота в Telegram (BotFather)

1. Відкрийте Telegram і в пошуку знайдіть **@BotFather** (офіційний бот з синьою галочкою).
2. Натисніть **Start** (або напишіть `/start`).
3. Надішліть команду `/newbot`, щоб створити нового бота.
4. BotFather запитає в вас 2 речі:
   - **Name:** Назва бота, яку бачитимуть користувачі (наприклад, `AI Community Local Bot`).
   - **Username:** Унікальний юзернейм, який обов'язково має закінчуватися на `bot` (наприклад, `nm_ai_community_local_bot`).
5. Після успішного створення BotFather надішле вам довге повідомлення. Скопіюйте **HTTP API Token** (виглядає як `123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ`).

---

## Крок 2. Налаштування OpenClaw (.env)

Тепер потрібно передати цей токен вашому локальному сервісу OpenClaw.

1. Відкрийте файл `docker/openclaw/.env` у вашому редакторі.
2. Додайте або оновіть наступні змінні:

```env
# Токен вашого нового телеграм-бота
TELEGRAM_BOT_TOKEN=123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ

# Для локальної розробки (Polling) залишаємо поле порожнім!
# OpenClaw буде сам "ходити" в Телеграм за оновленнями.
TELEGRAM_WEBHOOK_URL=
```

---

## Крок 3. Налаштування промпту OpenClaw (Системні інструкції)

Щоб бот знав, що робити, OpenClaw має бути налаштований діяти як оркестратор.
Якщо конфігурація OpenClaw зберігається у `.env` чи `config.yaml`, переконайтеся, що є базова інструкція (System Prompt) приблизно такого змісту:

> "Ти головний AI-асистент спільноти. Твоя ціль - розуміти наміри користувачів з Telegram і викликати профільних агентів (наприклад, hello-world, knowledge-agent), щоб отримати для них відповідь, після чого повернути фінальний результат користувачу."

_(Якщо OpenClaw налаштовується через UI (наприклад, веб-інтерфейс на локальному порту), зайдіть туди і додайте Telegram-вузол/канал та вставте туди System Prompt)._

---

## Крок 4. Запуск та тестування (Метод Polling)

Оскільки ми залишили `TELEGRAM_WEBHOOK_URL` порожнім, система мала б працювати в режимі **Long-polling** (внутрішній механізм постійно опитує Telegram).

1. Перезапустіть контейнер OpenClaw, щоб він підхопив новий `.env`:
   ```bash
   docker compose restart openclaw
   ```
2. Подивіться логи, щоб переконатись, що він підключився до Telegram без помилок:
   ```bash
   docker compose logs -f openclaw
   ```
3. Відкрийте ваш бот у Telegram за його юзернеймом (із Кроку 1).
4. Натисніть **Start** (надішлеться `/start`).
5. Напишіть своє перше повідомлення: **"Привіт, бот! Що ти хочеш сказати цьому світу?"**

### Що має відбутися під капотом:

1. Повідомлення потрапить в OpenClaw.
2. OpenClaw через A2A (дізнавшись про список доступних агентів) звернеться до `hello-world` агента, якщо він вже запущений `docker compose up -d hello-agent` і працює.
3. Ви отримаєте відповідь в Telegram.

---

## Альтернатива (Якщо OpenClaw вимагає тільки Webhook)

Якщо архітектура OpenClaw не підтримує Long-polling і йому **терміново** потрібна публічна адреса (Webhook Endpoint) для отримання повідомлень від Telegram:

1. Встановіть [ngrok](https://ngrok.com/) (безкоштовна тунельна утиліта).
2. Відкрийте тунель до порту, на якому висить Traefik або сам OpenClaw (наприклад, 80 або 8080):
   ```bash
   ngrok http 80
   ```
3. `ngrok` видасть вам публічну URL (наприклад: `https://abcd-123.ngrok-free.app`).
4. Скопіюйте її та вставте в `docker/openclaw/.env`:
   ```env
   TELEGRAM_WEBHOOK_URL=https://abcd-123.ngrok-free.app/webhook/telegram-path
   ```
5. Перезапустіть: `docker compose restart openclaw` (або `up -d`). OpenClaw при старті зареєструє цей ваш публічний ngrok-URL у Telegram.
6. Всі повідомлення з Telegram тепер летітимуть в ngrok -> на ваш локальний комп'ютер -> в Traefik -> в OpenClaw.
